# 🎬 Read Any Video

**让 LLM "看见"视频。**

大模型读不了 `.mp4`——本工具把视频拆成它能读的东西：
**关键帧图片 + 带时间戳的运动时间线**，
于是模型就能描述、总结、定位视频里的任意瞬间。

## 痛点

LLM 无法直接播放或理解视频。常见的"暴力抽帧"做法又会产生几百张几乎一样的图片，
既烧 token 又淹没真正有信息量的瞬间。

**Read Any Video** 用帧间差分分析先找出**哪里有动作**，
只在活跃片段抽帧，把算力和上下文都花在刀刃上：

- 🎯 **运动感知抽帧** —— 静止段折叠，活跃段才抽，信噪比高
- 🔍 **先扫描后聚焦** —— `scan` 看全局，`zoom` 抠细节，层层逼近
- 🧩 **九宫格总览** —— `grid` 一张图速览整段视频
- 📦 **零额外依赖** —— 仅需一个 Python 脚本，OpenCV / NumPy 自动安装
- 🤖 **为 Agent 而生** —— stderr 给人看的时间线，stdout 给模型读的 JSON

## 🔬 工作原理

```
视频 ──► 降采样 + 灰度化每一帧
      ──► 计算相邻帧的差分得到「运动分数」
      ──► 自动阈值切分为 静止 / 活跃 片段
      ──► 静止段折叠丢弃，活跃段按密度抽帧
      ──► 输出 关键帧 + 时间线
```

核心思路：**变化即信息**。画面没动的地方对模型理解几乎没有增量，
直接折叠；画面在动的地方才是关键瞬间，按需加密抽帧。

## 🤖 作为 Agent Skill

本仓库同时是一个标准的 **Agent Skill**（见 [`SKILL.md`](./SKILL.md)）。
接入支持 Skill 的 Agent 后，只要用户给出视频文件 / URL 并要求描述、
总结或定位某个瞬间，Agent 便会自动调用本工具完成"看视频"。

设计要点：
- **双通道输出** —— stderr 给人看时间线，stdout 给模型读结构化 JSON
- **先扫描后聚焦** —— 避免一次性抽取几百帧淹没上下文
- **沙箱友好** —— 线程数固定为 1，帧输出到 `/tmp`，读完即弃
