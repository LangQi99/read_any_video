# 🎬 Read Any Video

**让 LLM "看见"视频。**

大模型读不了 `.mp4`——本工具把视频拆成它能读的东西：
**关键帧图片 + 带时间戳的运动时间线**，
于是模型就能描述、总结、定位视频里的任意瞬间。

[快速开始](#-快速开始) · [命令](#-命令) · [工作原理](#-工作原理) · [作为 Agent Skill](#-作为-agent-skill) · [参数速查](#-参数速查)

## 痛点

LLM 无法直接播放或理解视频。常见的"暴力抽帧"做法又会产生几百张几乎一样的图片，
既烧 token 又淹没真正有信息量的瞬间。

**Read Any Video** 用帧间差分分析先找出**哪里有动作**，
只在活跃片段抽帧，把算力和上下文都花在刀刃上：

- 🎯 **运动感知抽帧** —— 静止段折叠，活跃段才抽，信噪比高
- 🔍 **先扫描后聚焦** —— `scan` 看全局，`zoom` 抠细节，层层逼近
- 🧩 **九宫格总览** —— `grid` 一张图速览整段视频
- 📦 **零额外依赖** —— 仅需一个 Python 脚本，OpenCV / NumPy 自动安装
- 🤖 **为 Agent 而生** —— stderr 给人看的时间线，stdout 给模型读的 JSON

## 🚀 快速开始

```prompt
安装这个skill：https://github.com/LangQi99/read_any_video
```

## 🔬 工作原理

```
视频 ──► 降采样 + 灰度化每一帧
      ──► 计算相邻帧的差分得到「运动分数」
      ──► 自动阈值切分为 静止 / 活跃 片段
      ──► 静止段折叠丢弃，活跃段按密度抽帧
      ──► 输出 关键帧 + 时间线
```

核心思路：**变化即信息**。画面没动的地方对模型理解几乎没有增量，
直接折叠；画面在动的地方才是关键瞬间，按需加密抽帧。

## 📋 输出格式

模型读取关键帧后，产出带时间戳的客观叙事：

```
0–3s    蓝色毯子中的人，背对镜头，背景是建筑物
4.5–7s  镜头跟随移动，蓝天下是光秃秃的树
9–10s   人物转身，露出侧脸，远处有其他行人
```

## 🤖 作为 Agent Skill

本仓库同时是一个标准的 **Agent Skill**（见 [`SKILL.md`](./SKILL.md)）。
接入支持 Skill 的 Agent 后，只要用户给出视频文件 / URL 并要求描述、
总结或定位某个瞬间，Agent 便会自动调用本工具完成"看视频"。

设计要点：
- **双通道输出** —— stderr 给人看时间线，stdout 给模型读结构化 JSON
- **先扫描后聚焦** —— 避免一次性抽取几百帧淹没上下文
- **沙箱友好** —— 线程数固定为 1，帧输出到 `/tmp`，读完即弃

## ⚙️ 参数速查

**通用参数**（三个命令均适用）

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `--start` / `--end` | 全片 | 处理的时间区间（秒） |
| `--outdir` | 临时目录 | 帧 / 图片输出目录 |
| `--quality` | `70` | JPEG 质量（1–100） |
| `--max-width` | `900` | 抽帧最大宽度（像素），超出则等比缩放 |

**`scan` 专属**

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `--density` | `2` | 活跃段抽帧密度（帧/秒） |
| `--analyze-fps` | `10` | 运动分析的采样帧率 |
| `--threshold` | 自动 | 活跃 / 静止判定阈值，留空自动估算 |
| `--grid` | `16` | 运动签名的降采样网格边长 |

**`zoom` 专属**

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `--density` | `8` | 抽帧密度（帧/秒），通常高于 scan |

**`grid` 专属**

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `--rows` / `--cols` | `3` × `3` | 九宫格行列数 |
| `--cell-width` | `320` | 每格宽度（像素） |

## 📝 注意事项

- 依赖 `opencv-python-headless` + `numpy`，首次运行自动安装
- 线程数固定为 1 以兼容沙箱环境，**请勿移除脚本顶部的相关设置**
- 帧默认输出到 `/tmp`，属临时文件，读完即可丢弃
- **伪装文件**（`.txt` / `.bin` / `.zip`）：先解压或重命名再用——OpenCV 按内容而非扩展名识别
- 始终先 `scan` / `grid`，**切勿一次性暴力抽取几百帧**
